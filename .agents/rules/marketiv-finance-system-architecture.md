# Marketiv Finance Development Rules

Tech Stack:

- React
- Vite
- TypeScript
- TailwindCSS v4
- Supabase

Do not suggest:

- Next.js
- Prisma
- Redux
- Zustand

unless explicitly requested.

Architecture Principles:

- Keep implementation simple.
- Prioritize rapid MVP delivery.
- Prefer Supabase as the primary backend solution.
- Use Supabase Auth for authentication.
- Use Supabase Storage for file uploads.
- Use Supabase PostgreSQL as the database.

Code Style:

- TypeScript only.
- Reusable components.
- Avoid overengineering.
- Keep folder structure simple.

Business Context:

This project is an internal financial management system for Marketiv.

Core Modules:

- Dashboard
- Transactions
- RAB
- Reports
- Audit Trail
- User Management

Important Business Logic:

Transactions must be linked to a Detailed RAB item.

Detailed RAB is the source of truth for budgeting.

Transaction updates should affect:

- Used Amount
- Realization
- Dashboard Metrics

Design Requirements:

Follow the Marketiv Studio design language already generated by Figma Make.

Preserve existing UI whenever possible.
